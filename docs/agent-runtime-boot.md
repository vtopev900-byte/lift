# P-000 / Agent Runtime Boot

## Что это

Операция входа агента в Project_OS_LIFT.

Агент не просто читает документы. Агент должен создать рабочее состояние сессии, включить checkpoint и начать вести нижний контур фиксации.

## Зачем нужно

Без этой операции новый чат превращается в обычного ассистента: он понимает правила, но не исполняет их.

## Минимальный результат входа

- Routing View открыт.
- P0-правила извлечены.
- Runtime-state создан.
- Write mode объявлен как hybrid_auto_log.
- Event buffer открыт.
- Boot checkpoint создан.
- Технический след входа записан или выведен fallback.

## Auto

Технические события нижнего контура агент пишет сам, если инструмент записи доступен.

## Approval

Source of truth, архитектура, active process и accepted decisions меняются только после явного approval пользователя.
